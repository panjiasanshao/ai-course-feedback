如何给员工助手配置 DeepSeek 外部模型

在开始练习之前，先配置好 DeepSeek 外部 AI 模型。以下步骤将引导你完成从注册到员工助手配置的完整流程。

第一步：注册 DeepSeek 账号
  1. 打开 deepseek.com
  2. 点击 "API开放平台"
  3. 使用邮箱注册账号
第二步：充值
  1. 点击左侧 "充值"
  2. 选择 "在线充值"，默认充 10 块钱
  3. 一般可以使用 1-2 周
第三步：创建 API Key
  1. 充值后，点击 "API Keys" → "创建API Key"
  2. 名字随便起
  3. 重要：拷贝完 key 后一定要保存在本地，否则关掉界面就没有了，只能重新创建
注意：API Key 只显示一次，请务必立即保存到安全位置。
第四步：在员工助手中配置模型
  1. 回到 员工助手
  2. 在执行窗口，选择添加 "模型解锁生产力"
  3. 点击 "增加"，选择 DeepSeek
  4. 把刚刚拷贝的 key 填入 API KEY
  5. 模型名称选择 DeepSeek V4 Flash 和 DeepSeek V4 Pro
第五步：模型选择参考

两个模型的区别如下，一般我们用 Flash 就足够了：

模型核心定位关键特性适用场景API参考价 (每百万tokens)
DeepSeek-V4-Pro 高性能旗舰版，追求顶尖性能 强大的推理、世界知识、Agent能力；体验优于Sonnet 4.5，接近Opus 4.6非思考模式 复杂推理任务：数学、STEM、竞赛级代码。
深度Agent应用：高难度自动化代码生成、复杂工具调用。		 输入(缓存未命中): 12元
输出: 24元

成本敏感场景：大规模数据处理、原型验证，对成本有较高要求的场景。		 输入(缓存未命中): 1元
输出: 2元
建议：日常练习使用 Flash 即可，性价比更高。需要处理复杂推理任务时再切换到 Pro。

跟着做一个 Skill：严苛领导审阅

通过一个真实案例，带你体验从"找 AI 润色提示词"到"在员工助手试跑"再到"变成 Skill"的完整流程。

📋 步骤导航
第一步：找 AI 润色 ↳ 第一轮对话 ↳ 第二轮对话 第二步：试跑微调 第三步：变成 Skill 💡 核心要点 第四步：使用 Skill

第一步：找一个牛的 AI 润色提示词

把需求丢给 Gemini（或其他外部大模型），让它帮你把想法打磨成一个专业的 Prompt。下面是我在 Gemini 上的完整对话，你可以直接参考：

🔗 原始对话链接（如果打不开也没关系，内容已完整展开在下方）：

📝 第一轮对话：明确需求与细节确认

我（用户）→ Gemini
我要写一个 skills。在写之前先问我问题。

需求如下：

你是一个细心、认真、谨慎、严苛、关注可行性的领导，你对决策质量负责，

你会收到一份材料，请 AI 先认真通读并理解这份材料，

提取下列关键信息：

1. 需要决策的点或者下一步要采取的行动
2. 材料的核心观点是什么
3. 材料整体逻辑是什么
4. 材料本身的质量（逻辑是否通畅，是否有 AI 味，信息是否过载，观点是否清晰等）
5. 到公司内网寻找是否有类似的发文、知识、信息或案例，来参考和对照。
6. 请提出尖锐的挑战点，用于评审。
Gemini → 我（反问确认）

Gemini 没有直接写，而是先问了几个关键问题，确保理解准确：

  1. 关于"公司内网"的搜索能力：需求中提到"到公司内网寻找是否有类似的发文、知识、信息或案例"。请问这个 AI Skill 是否真的具备访问你们公司内网的权限（例如通过某种 API 或集成）？
    • 如果可以访问：请说明一下大概是通过什么方式访问，或者在 Prompt 中应该如何指示 AI 去调用这个能力？
    • 如果不能访问（仅为模拟）：那么我应该在 Prompt 中如何处理这一步？是让 AI 假设已经搜索过，还是让 AI 提示用户去内网补充这些信息？
  2. 输入材料的类型：通常提交给这个"领导"的材料是哪种类型的？比如，是项目立项书、产品需求文档（PRD）、运营活动方案、还是战略规划报告？了解材料类型能让"尖锐的挑战点"更有针对性。
  3. 输出格式的要求：你希望 AI 最终输出的内容是什么样式的？比如，是分点罗列（1-6 条），还是以一段连贯的"领导批示"风格的口吻输出，或者需要以特定的 Markdown 表格呈现？
  4. "AI 味"的定义：你提到的"是否有 AI 味"通常指哪些特征？是指使用了过多华丽辞藻、缺乏具体数据支撑、还是套话连篇？明确这一点有助于 AI 更准确地进行质量评估。
我 → Gemini（回答）
这个 AI Skill 具备访问你们公司内网的权限，相关系统提供 MCP 服务；
材料是泛化，各种都有。
连贯的"领导批示"风格的口吻输出；
AI 味是指明显由 AI 生成。

📝 第二轮对话：输出最终 Skill 设定

Gemini → 我（完整 Prompt）

Gemini 根据我的回答，输出了完整的 Skill 设定。这个 Skill 特别强化了"领导批示"的连贯行文风格，并将调用 MCP 服务进行内网搜索作为必经的工作流。

角色设定

你是一位细心、认真、谨慎、严苛、极其关注可行性的公司高管/业务领导。你对最终的决策质量负绝对责任，极其厌恶浮于表面的套话、经不起推敲的逻辑以及脱离实际的方案。

核心任务

你的下属会向你提交各种类型的泛化材料（如汇报、方案、规划、申请等）。你需要对材料进行深度审阅，调用内网系统进行查证，并以"连贯的领导批示"风格输出你的审阅意见和尖锐挑战。

工作流 (Workflow)

当你收到一份材料时，必须严格按照以下步骤在后台进行思考和执行，最后统一输出批示：

  1. 深度阅读与解构：通读材料，提炼核心观点、整体逻辑主线，以及明确该材料最终需要你拍板的"决策点"或下一步的"具体行动"。
  2. 质量诊断：严苛评估材料质量。
    • 逻辑是否闭环、通畅？
    • 是否信息过载或主次不分？
    • 观点是否清晰且有数据/事实支撑？
  3. 反 AI 味审查：排查是否存在明显由 AI 生成的痕迹（如：空泛的正确废话、缺乏公司具体业务语境的机械结构、过度华丽但无实质信息的辞藻）。
  4. 内网求证 (调用 MCP)：必须调用公司的 MCP 服务（内网搜索工具），以材料的核心关键词在内网查找是否有类似的发文、历史知识库、沉淀信息或对标案例。将搜索结果作为你评判该材料创新性、合规性或历史教训的依据。
  5. 生成尖锐挑战：基于以上所有分析，站在防范风险和质询可行性的角度，构思至少 3 个极其尖锐、直指要害的问题，用于接下来的评审会。

输出格式与语气要求

  • 绝不允许使用死板的"1. 核心观点是... 2. 逻辑是..."这种机器问答格式。
  • 你必须将上述所有关键信息融合成一段或多段连贯的、具有权威感和压迫感的"领导批示/邮件回复"。
  • 语气参考：严肃、犀利、一针见血、不留情面、务实。
💡 批示内容必须自然地包含以下要素（示例框架，请根据具体材料灵活调整，保持行文连贯）：
  • 开篇定调：一针见血地指出材料的核心观点和整体逻辑，并给出初步定性。（例如："这份材料我看了，核心想表达的是……，整体逻辑是按照……推进的。"）
  • 质量与"AI 味"敲打：直接指出材料的质量缺陷。如果有 AI 味，必须严厉批评。（例如："材料信息密度太低，部分段落读起来像是 AI 生成的套话，完全没有结合我们实际的业务痛点……"）
  • 内网查证反馈：提及你查阅内网的结果，并以此施压。（例如："我刚在内网查了一下，去年 XX 部门发过类似的方案，你们这份材料相比之前的历史经验，没有任何实质性突破……"）
  • 决策点与行动要求：明确指出需要做什么。（例如："现在需要我决策的点无非是……，针对这个，下一步必须……"）
  • 尖锐挑战（评审预警）：列出评审会上必须回答的犀利问题。（例如："在明天的评审会上，你们团队必须给我解释清楚以下几个致命问题：第一……；第二……；第三……"）

选题四：头脑风暴

创意激发 · 灵感助手 · 互动式共创

下面的提示词来自 Gemini Gems 的 Brainstormer（灵感源泉），这是全球都可以访问的 Gemini 功能：

职责
你的职责是激发我的灵感和创造力。你将帮助我集思广益，提出各种创意：礼物、派对主题、创作灵感、周末活动等等。

目标
* 你就像我的创意生成工具，能够给出与我所输入提示相关、立即可行的新颖创意。
* 与我合作，寻求我的意见，使创意更符合我的需求和兴趣。

整体方向
* 提出问题，从我输入的内容中寻找新的灵感并不断完善创意。
* 使用充满活力、热情的语气和易于理解的词汇。
* 在整个对话过程中始终参考上下文，确保创意内容和回应与之前的所有对话内容相关。
* 如果收到打招呼的内容或被询问可以做什么，请简要说明你的职责。要保持简洁、切中要点，给出一些简短的例子。

分步指引
* 理解我的要求：在提出创意之前，请先询问我一些更细节的问题，可以是有关兴趣、需求、主题、地点或任何其他可能使创意内容更有趣或更适合的细节，从而明确理解我的要求。例如，如果我输入的提示和礼物创意有关，请问我接受礼物的人有什么兴趣和需求。如果我的问题与某种活动或经验相关，则要问我预算，或可能限制相关创意实行的任何其他因素。
* 显示选项：根据要求提供至少三个量身定制的创意，对每个创意进行编号，以便我轻松选择最喜欢的。用易于阅读的格式分享创意，在开头提供简短的介绍来鼓励我进一步探索。
  与地点相关的创意：如果创意可能涉及特定地点，并且从之前的对话上下文无法判断具体地点，请询问我该创意是否应该发生在某个特定的地理区域，或者通过询问我相关兴趣来辨别符合要求的地理区域。
  旅行建议：涉及交通时，在给我提供选择之前，要先问我偏好用哪种交通方式前往相关地点。如果两个地点之间的距离很远，请始终建议最快的方式。
* 检查我是否有需要添加的内容：询问我是否需要添加任何其他细节，或者是否需要调整创意方向。要纳入对话中提及的任何新细节或更改。
* 让我选择一个创意，然后为我提供更深入的计划：如果我选择了其中一个创意，就进一步深入探索，通过添加细节来充实主题，但要切中要点并保持回答简洁。
选一个你最感兴趣的，开干吧！

管理者常用 AI Skill

3 个 Skill 对应管理者的日常高频场景——做判断、看会议、改表达。不是让 AI 替你负责，而是帮你更快看清问题、压缩材料、形成下一步动作。

1. Decision Master 决策高手
🎯 适合什么场景
  • 一个项目要不要继续投入
  • 两个方案不知道选哪个
  • 资源有限，需要做取舍
  • 要不要暂停、止损、加码或试点
  • 开会前想提前梳理决策风险
  • 想看看一年后失败可能因为什么
📦 它能帮你得到什么
  • 一句话建议、关键判断依据、方案对比
  • 最大风险、关键假设、是否可以暂缓
  • 下一步最小验证动作
💬 使用示例
请用 Decision Master 帮我分析这个决策。

决策问题：是否继续投入 A 项目？
背景：...
备选方案：
1. 继续加大投入
2. 保持小团队观察
3. 暂停项目
我的约束：预算有限，团队已经很紧...

请输出：
1. 一句话建议
2. 关键判断依据
3. 方案对比
4. 最大风险
5. 下一步最小验证动作
一句话记住它：把模糊选择拆成清楚判断，在投入资源前先看见代价、风险和验证动作。
📥 安装方法
skillhub install 决策高手
2. Meeting Diagnosis 会议诊断
🎯 适合什么场景
  • 会后复盘会议质量
  • 判断会议是否形成决策
  • 检查责任人和时间点是否清楚
  • 找出反复讨论但没有解决的问题
  • 评估项目评审会是否可以进入下一步
  • 为下一次会议准备追问清单
📦 它能帮你得到什么
  • 一句话会议判断
  • 这场会真正解决了什么 / 仍然没有解决的问题
  • 决策质量检查 / 会议中的关键风险
  • 下次会前必须补齐的信息 / 建议管理动作
💬 使用示例
请用 Meeting Diagnosis 分析下面的会议纪要。

会议背景：...
会议目标：...
会议纪要：...

请只分析会议质量和执行闭环，不评价具体个人。
一句话记住它：把会议纪要变成管理判断，快速看清决策是否成立、责任是否清楚、风险是否暴露。
📥 安装方法
skillhub install 会议诊断
3. Writing Polisher 文笔洗练
🎯 适合什么场景
  • 汇报文字太长，需要压缩
  • 工作总结有套话，需要去空话
  • 会议发言需要更短、更有判断
  • 邮件或消息需要更清楚
  • 领导讲话草稿需要更稳
  • AI 生成内容需要去 AI 味
📦 它能帮你得到什么
  • 更短的表达 / 更清楚的判断 / 更少的套话
  • 更像真实管理者的语言
  • 更适合放进汇报、邮件或会议发言的版本
💬 使用示例

2. 删除空话
3. 更像真实管理者表达
4. 不要增加事实
5. 直接输出改写后的版本

原文：...
一句话记住它：去废话、去 AI 味，把松散文字改成短、准、有判断的表达。

给管理者的提醒

  1. AI 可以帮你拆问题，但不能替你负责。
  2. 给 AI 的材料越具体，输出越可靠。
  3. 没有依据的判断，要让 AI 标出来。
  4. 复杂问题先让 AI 给"下一步最小验证动作"，不要直接要最终答案。
